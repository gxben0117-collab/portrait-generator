# 美片咒語產生器 - 專案說明文件

## 📋 專案概述

這是一個靜態網頁應用程式，用於生成電影級人像攝影的 AI 咒語（Prompt）。系統採用 7 步驟咒語生成順序和 13-tier Prompt Governance 過濾系統，確保生成真實、專業的人像攝影效果。

**線上網址：** https://gxben0117-collab.github.io/portrait-generator/

## 🔑 重要檔案說明

### 1. **核心資料/風格範例.md** ⭐ 最重要
- **作用：** 唯一數據源，所有角色卡資料都存放在這裡
- **格式：** Markdown 格式，易於人工編輯
- **內容：** 758 個角色卡，384 個分類，22 個風格大類
- **編輯：** 新增或修改角色卡時，直接編輯此檔案
- **建置：** 編輯後執行 `npm run build` 生成 HTML

### 2. **index.html**
- **作用：** 前端主頁面，包含完整的 UI 和 CATS 陣列
- **生成：** 由 `rebuild_with_parent_categories.mjs` 自動生成
- **注意：** ⚠️ 不要手動編輯此檔案，所有修改會在下次建置時被覆蓋

### 3. **core.js**
- **作用：** 核心咒語生成邏輯
- **功能：**
  - 7 步驟咒語生成順序
  - 智能判斷是否使用擴展版身體比例描述
  - 真人身份優先機制
  - 13-tier Prompt Governance 過濾

### 4. **scripts/rebuild_with_parent_categories.mjs** ⭐ 主要建置腳本
- **作用：** 從 `風格範例.md` 讀取資料，生成 `index.html` 的 CATS 陣列
- **功能：**
  - 解析 Markdown 格式的角色卡
  - 根據關鍵字自動歸類到 22 個風格大類
  - 生成 384 個詳細分類
  - 更新 HTML 中的 CATS 和 PARENT_CATEGORIES 陣列

### 5. **核心資料/核心咒語規範.md**
- **作用：** 咒語生成規範與欄位說明
- **內容：** 各個欄位的定義、用途、可選值

### 6. **核心資料/角色卡建檔指南.md**
- **作用：** 角色卡格式規範
- **內容：** 如何新增角色卡、欄位說明、禁止詞彙清單

## 🔄 工作流程

### 新增角色卡流程

1. **編輯數據源**
   ```bash
   # 打開風格範例.md，按照格式新增角色卡
   notepad "核心資料/風格範例.md"
   ```

2. **執行建置**
   ```bash
   npm run build
   ```

3. **測試驗證**
   ```bash
   npm run dev
   # 在瀏覽器打開 http://localhost:8080
   ```

4. **提交推送**
   ```bash
   git add -A
   git commit -m "feat: 新增 XXX 角色卡"
   git push origin main
   ```

5. **自動部署**
   - GitHub Actions 會自動部署到 GitHub Pages
   - 約 1-2 分鐘後網站更新

### 修改現有角色卡流程

1. 在 `風格範例.md` 中找到對應的角色卡
2. 修改內容
3. 執行 `npm run build`
4. 測試並提交

## 🎯 角色卡格式規範

### Markdown 格式

```markdown
#### 角色卡名稱
- **ID:** `unique_id_01`
- **分類:** 詳細分類名稱
- **妝容：** makeup_style
- **場景背景：** scene description in English...
- **光線：** ls_cinematic
- **妝容描述：** makeup description in English...
- **服裝：** outfit description in English...
- **道具：** prop description in English...
- **構圖：** composition description in English...
- **妝容關鍵字：** oriental / western / fantasy
- **模板：** default / minimal
- **比例：** r_916 / r_45 / r_34 / r_11 / r_169
- **鏡頭：** l_35 / l_50 / l_85 / l_135
- **角度：** sanfen / front / side
- **鏡頭語言：** cl_documentary / cl_cinematic / cl_fashion
- **氛圍：** at_clear / at_soft / at_dramatic
```

### 欄位說明

| 欄位 | 必填 | 說明 | 範例 |
|------|------|------|------|
| ID | ✅ | 唯一識別碼 | `han_01`, `myth_01` |
| 分類 | ✅ | 詳細分類名稱 | 漢代系列、魅魔系列 |
| 妝容 | ✅ | 妝容風格代碼 | `hanfu_elegant` |
| 場景背景 | ✅ | 場景描述（英文） | golden hour desert... |
| 光線 | ✅ | 光線類型 | `ls_cinematic` |
| 妝容描述 | ✅ | 妝容詳細描述（英文） | natural makeup... |
| 服裝 | ✅ | 服裝描述（英文） | layered robe... |
| 道具 | ⚠️ | 道具與構圖（英文） | woman in close-up... |
| 構圖 | ⚠️ | 構圖描述（英文） | upper-body framing... |
| 妝容關鍵字 | ✅ | 妝容類型 | oriental / western / fantasy |
| 模板 | ✅ | 模板類型 | default / minimal |
| 比例 | ✅ | 輸出比例 | r_916 / r_45 / r_34 |
| 鏡頭 | ✅ | 鏡頭焦距 | l_35 / l_50 / l_85 |

| 鏡頭語言 | ✅ | 鏡頭風格 | cl_documentary |

## 🏗️ 7 步驟咒語生成順序

1. **鎖臉（STEP 1）** - 真人身份優先
2. **身體比例（STEP 2）** - 真實人體骨架（標準版/擴展版）
3. **姿勢（STEP 3）** - 自然姿勢與動作
4. **背景服飾道具（STEP 4-6）** - 場景、服裝、道具
5. **鏡頭規格（STEP 7）** - 鏡頭、角度、光線、構圖
6. **禁止咒語（STEP 8）** - 13-tier 過濾系統
7. **最終覆蓋（STEP 9）** - 最終強化指令

## 🛠️ NPM 腳本命令

```json
{
  "build": "node scripts/rebuild_with_parent_categories.mjs",
  "sync": "node scripts/sync_cats_to_md.mjs",
  "add-categories": "node scripts/add_category_to_md.mjs",
  "dev": "python -m http.server 8080",
  "test": "node scripts/test.mjs"
}
```

### 命令說明

- **`npm run build`** - 從 `風格範例.md` 生成 `index.html`（最常用）
- **`npm run dev`** - 啟動本地開發伺服器（http://localhost:8080）
- **`npm run sync`** - 從 `index.html` 同步到 `風格範例.md`（備用）
- **`npm run add-categories`** - 為角色卡添加分類標記（備用）
- **`npm run test`** - 執行測試腳本

## ⚠️ 注意事項

### 1. 數據源管理
- ✅ **唯一數據源：** `風格範例.md` 是唯一的數據來源
- ❌ **不要手動編輯：** `index.html` 中的 CATS 陣列會被自動覆蓋
- ✅ **編輯流程：** 編輯 `.md` → 執行 `npm run build` → 生成 HTML

### 2. ID 命名規則
- 使用有意義的前綴：`han_01`（漢代）、`myth_01`（神話）、`scifi_01`（科幻）
- 確保 ID 唯一，不要重複
- 使用小寫字母和底線

### 3. 分類命名
- 使用清晰、具體的分類名稱
- 避免過於籠統的分類（如"其他"、"混合"）
- 分類名稱會影響自動歸類到風格大類

### 4. 英文描述
- 場景、服裝、道具、構圖等欄位使用英文描述
- 使用專業攝影術語
- 避免使用禁止詞彙（參考 `角色卡建檔指南.md`）

### 5. 擴展版身體比例
以下風格會自動使用擴展版身體比例描述，避免過度性感化：
- 魅魔、黑暗女王、哥德、暗黑、西域
- 歐美奇幻、暗黑電影、女魔王、墮天使
- 吸血鬼、夜之女王、莉莉絲、魔冕、冥界
- 滅世、暗夜、墮落、魔女、女巫、深淵、魔域

### 6. Git 工作流程
- 每次修改後記得提交到 Git
- 推送到 GitHub 後會自動部署到 GitHub Pages
- 保持提交訊息清晰明確

### 7. 備份管理
- 重要修改前先備份 `風格範例.md`
- 備份檔案放在 `核心資料/` 目錄下
- 定期清理舊備份

## 🐛 常見問題

### Q1: 為什麼選擇風格大類後，場景主題不對？
**A:** 這是因為分類映射關係錯誤。解決方法：
1. 檢查 `風格範例.md` 中角色卡的 `- **分類:**` 欄位是否正確
2. 執行 `npm run build` 重新生成
3. 如果問題持續，檢查 `rebuild_with_parent_categories.mjs` 的關鍵字規則

### Q2: 新增角色卡後網頁沒有顯示？
**A:** 確認以下步驟：
1. 是否執行了 `npm run build`？
2. 是否刷新了瀏覽器快取（Ctrl+F5）？
3. 檢查 `風格範例.md` 格式是否正確

### Q3: 如何刪除重複的角色卡？
**A:** 
1. 在 `風格範例.md` 中手動刪除重複項
2. 或執行 `node scripts/reorganize_by_theme.mjs` 自動去重

### Q4: 如何修改風格大類？
**A:** 編輯 `scripts/rebuild_with_parent_categories.mjs` 中的 `PARENT_CATEGORY_RULES` 陣列
