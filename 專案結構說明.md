## 🎯 核心檔案說明

### 1. 前端檔案

#### `index.html` ⭐ 主頁面
- **作用：** 前端主頁面，包含完整的 UI 和 CATS 陣列
- **生成方式：** 由 `rebuild_with_parent_categories.mjs` 自動生成
- **包含內容：**
  - 電影級 UI 設計
  - CATS 陣列（828 個角色卡，396 個分類）
  - PARENT_CATEGORIES 陣列（22 個風格大類）
  - 風格選擇介面
  - 咒語生成邏輯調用
- **⚠️ 注意：** 不要手動編輯此檔案，所有修改會在下次建置時被覆蓋

#### `core.js` ⭐ 核心邏輯
- **作用：** 核心咒語生成邏輯
- **功能：**
  - 7 步驟咒語生成順序
  - 智能判斷是否使用擴展版身體比例描述
  - 真人身份優先機制
  - buildPrompt() 主函數
- **可編輯：** ✅ 可以手動編輯以調整咒語生成邏輯

#### `prompt_governance.js`
- **作用：** 13-tier Prompt Governance 過濾系統
- **功能：** 禁止詞彙過濾、內容審查
- **可編輯：** ✅ 可以手動編輯以調整過濾規則

---

### 2. 數據檔案

#### `核心資料/風格範例.md` ⭐⭐⭐ 最重要
- **作用：** 唯一數據源，所有角色卡資料都存放在這裡
- **格式：** Markdown 格式，易於人工編輯
- **內容：**
  - 828 個角色卡
  - 396 個分類
  - 22 個風格大類（通過關鍵字自動歸類）
- **編輯流程：**
  1. 直接編輯此檔案新增或修改角色卡
  2. 執行 `npm run build` 生成 HTML
  3. 測試驗證
  4. 提交到 Git

#### `核心資料/核心咒語規範.md`
- **作用：** 咒語生成規範與欄位說明
- **內容：**
  - 7 步驟咒語生成順序說明
  - 各個欄位的定義、用途、可選值
  - 禁止詞彙清單
  - 最佳實踐建議

#### `核心資料/角色卡建檔指南.md`
- **作用：** 角色卡格式規範
- **內容：**
  - 如何新增角色卡
  - 欄位說明與範例
  - 禁止詞彙清單
  - 常見錯誤與注意事項

#### `核心資料/妝容模組.json`
- **作用：** 妝容資料模組
- **格式：** JSON 格式
- **內容：** 各種妝容風格的定義

---

### 3. 建置腳本

#### `scripts/rebuild_with_parent_categories.mjs` ⭐ 主要建置腳本
- **作用：** 從 `風格範例.md` 讀取資料，生成 `index.html` 的 CATS 陣列
- **功能：**
  - 解析 Markdown 格式的角色卡
  - 根據關鍵字自動歸類到 22 個風格大類
  - 生成 394 個詳細分類
  - 更新 HTML 中的 CATS 和 PARENT_CATEGORIES 陣列
- **使用：** `npm run build`

#### `scripts/build_cats_from_md.mjs`
- **作用：** 簡單版建置腳本（備用）
- **功能：** 基本的 Markdown 解析和 HTML 生成
- **使用：** 手動執行 `node scripts/build_cats_from_md.mjs`

#### `scripts/sync_cats_to_md.mjs`
- **作用：** 從 `index.html` 同步到 `風格範例.md`（備用）
- **功能：** 反向同步，從 HTML 提取資料寫入 Markdown
- **使用：** `npm run sync`

#### `scripts/add_category_to_md.mjs`
- **作用：** 為角色卡添加分類標記（備用）
- **功能：** 從 HTML 提取分類資訊並更新到 Markdown
- **使用：** `npm run add-categories`

#### `scripts/reorganize_by_theme.mjs`
- **作用：** 智能整理分類（備用）
- **功能：** 根據主題和場景重新分類，刪除重複項
- **使用：** 手動執行 `node scripts/reorganize_by_theme.mjs`

#### `scripts/test.mjs`
- **作用：** 測試腳本
- **功能：** 測試各種功能
- **使用：** `npm run test`

---

### 4. 配置檔案

#### `package.json`
- **作用：** NPM 配置與腳本命令
- **包含命令：**
  - `npm run build` - 建置專案

#### `README.md`
- **作用：** 專案簡介
- **內容：** 專案概述、快速開始、基本使用說明

#### `專案說明.md`
- **作用：** 完整專案說明文件
- **內容：** 詳細的專案說明、工作流程、欄位說明、注意事項

## 🔄 工作流程

### 新增角色卡

```bash
# 1. 編輯數據源
notepad "核心資料/風格範例.md"

# 2. 執行建置
npm run build

# 3. 測試驗證
npm run dev
# 在瀏覽器打開 http://localhost:8080

# 4. 提交推送
git add -A
git commit -m "feat: 新增 XXX 角色卡"
git push origin main

# 5. 自動部署（GitHub Actions）
# 約 1-2 分鐘後網站更新
```

### 修改現有角色卡

```bash
# 1. 在 風格範例.md 中找到對應的角色卡
# 2. 修改內容
# 3. 執行 npm run build
# 4. 測試並提交
```

### 修改咒語生成邏輯

```bash
# 1. 編輯 core.js
# 2. 測試驗證
# 3. 提交推送
```

## ⚠️ 重要注意事項

### 1. 單一數據源原則
- ✅ **唯一數據源：** `風格範例.md` 是唯一的數據來源
- ❌ **不要手動編輯：** `index.html` 中的 CATS 陣列會被自動覆蓋
- ✅ **編輯流程：** 編輯 `.md` → 執行 `npm run build` → 生成 HTML

### 2. 建置流程
- 每次修改 `風格範例.md` 後，必須執行 `npm run build`
- 建置腳本會自動生成 `index.html` 的 CATS 和 PARENT_CATEGORIES 陣列
- 不要跳過建置步驟直接提交

### 3. Git 工作流程
- 每次修改後記得提交到 Git
- 推送到 GitHub 後會自動部署到 GitHub Pages
- 保持提交訊息清晰明確

### 4. 備份管理
- 重要修改前先備份 `風格範例.md`
- 備份檔案放在 `核心資料/` 目錄下
- 定期清理舊備份
