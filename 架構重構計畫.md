# 架構重構計畫 - 角色卡數據管理

## 當前問題（2026-05-25）

### 數據重複維護問題
目前角色卡數據存在於兩個地方，需要手動同步：

1. **index.html** - 內嵌的 `const CATS = [...]` 陣列（923個角色卡）
2. **核心資料/風格範例.md** - Markdown 格式的角色卡文檔（923個角色卡）

**問題：**
- 新增或修改角色卡時需要同時更新兩個文件
- 容易造成數據不一致
- 維護成本高，容易出錯
- 例如：剛才添加 `cupSize` 欄位時只修改了 index.html，風格範例.md 並未同步

## 計畫方案：選項 1 - 單一數據源（JSON）

### 目標架構
```
cards.json (單一真相來源 - Single Source of Truth)
    ↓
    ├→ index.html 構建時注入或運行時載入
    └→ 風格範例.md 由腳本自動生成（僅供文檔參考）
```

### 實施步驟

#### 階段 1：提取數據到 JSON
- [ ] 從 index.html 提取 CATS 陣列
- [ ] 創建 `核心資料/cards.json` 作為主數據源
- [ ] 保持完整的數據結構（所有欄位）

#### 階段 2：修改前端載入方式
選擇以下其中一種方式：

**方式 A：構建時注入（推薦）**
- [ ] 創建構建腳本 `scripts/build.mjs`
- [ ] 讀取 `cards.json` 並注入到 index.html
- [ ] 生成最終的 `dist/index.html`

**方式 B：運行時動態載入**
- [ ] 修改 index.html，移除內嵌的 CATS 陣列
- [ ] 添加 `fetch('核心資料/cards.json')` 載入邏輯
- [ ] 注意：需要處理跨域問題（本地開發時）

#### 階段 3：自動生成文檔
- [ ] 創建腳本 `scripts/generate_style_doc.mjs`
- [ ] 從 `cards.json` 讀取數據
- [ ] 自動生成 `核心資料/風格範例.md`
- [ ] 在構建流程中自動執行

#### 階段 4：更新工作流程
- [ ] 更新 README.md 說明新的工作流程
- [ ] 添加 npm scripts：
  - `npm run build` - 構建 index.html
  - `npm run docs` - 生成風格範例.md
  - `npm run dev` - 開發模式（如果需要）
- [ ] 更新 `.gitignore`（如果 index.html 變成生成文件）

## 重要提醒給未來的 AI

### ⚠️ 修改角色卡數據時
1. **只修改 `核心資料/cards.json`** - 這是唯一的數據源
2. **不要直接修改 index.html 中的 CATS 陣列** - 它應該是自動生成的
3. **不要直接修改風格範例.md** - 它應該是自動生成的
4. 修改後執行：
   ```bash
   npm run build  # 重新生成 index.html
   npm run docs   # 重新生成風格範例.md
   ```

### 📝 添加新角色卡的流程
```bash
# 1. 編輯 cards.json，添加新角色卡
# 2. 重新構建
npm run build
npm run docs
# 3. 提交變更
git add 核心資料/cards.json index.html 核心資料/風格範例.md
git commit -m "feat: 添加新角色卡 XXX"
```

### 🔧 添加新欄位的流程（如 cupSize）
```bash
# 1. 使用腳本批量修改 cards.json
node scripts/add_cup_size.mjs
# 2. 更新 core.js 支持新欄位
# 3. 重新構建
npm run build
npm run docs
# 4. 更新文檔說明新欄位的用途
```
